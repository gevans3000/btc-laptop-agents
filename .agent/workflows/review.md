# Review

Goal: produce a concise, risk-focused code review report for recent changes.

Constraints: prioritize correctness, security, performance, edge cases, logging, error handling, and style. Reference file paths and line numbers when possible.

Output format:
1) Findings (ordered by severity, include file:line)
2) Risks/Tests missing
3) Change summary (brief)
